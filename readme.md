```

│   ├── grpcstreamout/                      #   gRPC client for StreamOut (lower layer)
│   │   ├── grpc_str_out.proto              #     Service definition
│   │   ├── StreamoutGrpcClientInterface.h  #     Abstract interface

│   │   └── StreamoutGrpcClient.cpp         #     Real gRPC client implementation
```

## Build

Build from the project root:

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Debug
cmake --build build
./build/test/unit_test
```

This builds `libmediacontroller.a` (static library) in `build/sources/` and the test binary in `build/test/`.

Requires: `protobuf-compiler`, `libprotobuf-dev`, `libgrpc++-dev`.

### Library Integration

The `sources/` folder builds as a static library (`libmediacontroller.a`). In the mk2 app, link against this library instead of compiling the sources directly. The library exports all necessary include paths and dependencies (protobuf, gRPC) via CMake's `PUBLIC` target properties.

### MEDIACONTROLLER_INTERFACE_DIR

The library needs `MediaControllerInterface.h`. By default it looks in `mock/`, so standalone builds work out of the box:

```bash
cmake -S . -B build                 # uses mock/ — no flag needed
```

When integrating into the mk2 app (or any other project that provides the real `MediaControllerInterface.h`), pass the actual path:

```bash
cmake -S . -B build -DMEDIACONTROLLER_INTERFACE_DIR=/path/to/mk2/includes
```

## Test Modes

Controlled by `#define` at the top of `test/unit_test.cpp`:

| Mode | Define | What it does |
|------|--------|--------------|
| Mock (unit tests) | Comment out `TEST_GRPC_RECONNECT` | Uses `StreamoutGrpcClientMock`, runs all asserts, exits |
| Reconnect (integration) | `#define TEST_GRPC_RECONNECT 1` | Uses real `StreamoutGrpcClient`, connects to target, waits for Enter to quit |

### Mock mode (default for CI)
```cpp
// #define TEST_GRPC_RECONNECT 1   // ← commented out
```
Runs all unit tests with mock gRPC client, no server needed.

### Reconnect mode (manual testing)
```cpp
#define TEST_GRPC_RECONNECT 1
```
Connects to the real gRPC server (`10.116.165.104:50052`). If server is down,
auto-reconnect logs appear every 5 seconds. Press Enter to quit.

## Key Concepts

| File | Generated by | Contains |
|------|-------------|----------|
| `.pb.h` / `.pb.cc` | `protoc --cpp_out` | Message/data classes (StatusRequest, HelloReply, etc.) |
| `.grpc.pb.h` / `.grpc.pb.cc` | `protoc --grpc_out` | Client stub + server base class (requires libgrpc++-dev) |

## Folder Layout

| Folder | Purpose |
|--------|---------|
| `mock/` | Headers/stubs for dependencies not yet available (provided by mk2 app at integration time) |
| `sources/` | Production code that will be used in the final mk2 app build |
| `sources/grpcstreamout/` | gRPC client for StreamOut lower layer |
| `sources/grpcstreamin/` | gRPC client for StreamIn (future) |
| `test/` | Test code for standalone module development and validation |

During standalone development, `mock/` headers are included via the library's `PUBLIC` include paths.
When integrated into the mk2 app build system, replace `mock/` with the real include paths.
